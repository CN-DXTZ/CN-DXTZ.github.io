准备

阅读官方主题文档：

布局模板决定了页面的结构，而通过回调，每个主题都包含了 index 模板，以下是各页面相对应的模板：

模板 页面 回调
index 首页
post 文章 index
page 分页 index
archive 归档 index
category 分类归档 archive
tag 标签归档 archive

Hexo 的生成原理也很简单，主要由 generator 和 renderer 构成：

  • generator 通过上述布局模板生成页面及控制目录结构：
  • renderer 负责渲染页面，（本文主要负责修改的部分，选择默认技术栈）：
    • ejs：即上述布局模板，可以简单理解为 JS + HTML
    • stylus：可以简单理解为 CSS

以下正式进行主题的制作

主题文件初始化

首先初始化主题文件，以下通过 yo 和 generator-hexo-theme 自动生成（亦可以手动新建各文件）：
先安装包，在命令行输入并执行：

npm install yo -g
npm install generator-hexo-theme -g

然后在命令行进入待创建主题的文件夹，输入并执行下列命令：

yo hexo-theme

并进行可选项选择，包括：主题名，技术栈，其他等，效果如下:

执行完毕后即主题文件初始化基本完成，再稍作修改：

  • 新建文件夹：/languages/
  • 修改文件夹名：/layout/partial/ 改为 /layout/_partial/
  • 新建文件夹： /source/fonts/
  • 新建文件夹： /source/images/ 并将 favicon.ico 移入

最终目录结构及对应的意义如下：

├── languages ### 语言支持
├── layout ### 布局模块文件夹 
│   ├── _partial ### 可重复利用的局部布局模块文件夹 
│   ├── archive.ejs ### 归档 ejs 布局 
│   ├── category.ejs ### 分类 ejs 布局 
│   ├── index.ejs ### 首页 ejs  ejs 布局 
│   ├── layout.ejs ### 主 ejs 布局 
│   ├── page.ejs ### 分页 ejs 布局 
│   ├── post.ejs ### 文章详情 ejs 布局 
│   └── tag.ejs ### 标签 ejs 布局
└── source ### 资源文件夹
│   ├── css ### css 文件夹 
│   │   └── 主题名.styl ### styl 渲染
│   ├── fonts ### 字体文件夹
│   ├── images ### 图片文件夹
│   │   └──  favicon.ico ### 网站图标 
│   └── js ## js 文件夹 
│       └── 主题名.js ### js 脚本 
└── _config.yml ### 主题的配置文件
